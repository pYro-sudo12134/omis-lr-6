# Описание проекта: Лабораторная работа №6 (Система анализа речевых запросов)

## Общая информация
Проект представляет собой веб-приложение для анализа речевых запросов с использованием многослойной архитектуры Java EE. Система предназначена для обработки, хранения и анализа речевых данных с интегрированными датчиками и звуковыми компонентами(скажем так, сделано на будущее).

## Архитектурные особенности

### 1. **Многослойная архитектура (N-Tier)**
Проект построен по классической многоуровневой архитектуре с четким разделением ответственности:

#### Уровни архитектуры:
- **Представление (Presentation Layer)**: JSP/HTML + Servlets
- **Бизнес-логика (Business Layer)**: Services + DTOs
- **Доступ к данным (Data Access Layer)**: Repositories + JPA
- **Модель данных (Domain Model)**: Entities + Enums

### 2. **Структура пакетов**

#### Основные модули:

```
src/main/java/by/losik/lab6omis/
├── entities/           # Сущности JPA (модель данных)
│   ├── base/          # Базовые классы сущностей
│   └── general/       # Основные сущности и перечисления
├── repository/        # Паттерн Repository для доступа к данным
│   ├── base/         # Базовый репозиторий
│   └── general/      # Специализированные репозитории
├── service/          # Бизнес-логика
│   ├── base/        # Базовые сервисы и стратегии
│   ├── cache/       # Кэширование данных
│   └── general/     # Сервисы предметной области
├── resource/         # REST API ресурсы (JAX-RS)
│   ├── base/        # Базовый ресурс
│   └── general/     # Специализированные ресурсы
├── servlet/          # Servlets для веб-интерфейса
├── dto/             # Data Transfer Objects
├── filter/          # Фильтры (аутентификация, кодировка)
└── exception/       # Обработчики исключений
```

### 3. **Технологический стек**

#### Основные технологии:
- **Java EE 8+** (JPA, CDI, JAX-RS)
- **JPA/Hibernate** для ORM
- **PostgreSQL** (предположительно, на основе SQL-файла)
- **Servlets/JSP** для веб-интерфейса
- **REST API** (JAX-RS)
- **EhCache** для кэширования
- **CDI** для dependency injection

### 4. **Ключевые паттерны проектирования**

#### a) **Repository Pattern**
- `BaseRepository<T>` - обобщенный базовый репозиторий
- Специализированные репозитории для каждой сущности

#### b) **Service Layer Pattern**
- `BaseService<T>` - базовый сервис с общими операциями
- Сервисы для бизнес-логики (анализ, обработка данных)

#### c) **DTO Pattern**
- Отдельные классы для передачи данных между слоями
- Специализированные DTO для разных сценариев использования

#### d) **Strategy Pattern**
- `AnalysisStrategy` - интерфейс стратегии анализа
- Конкретные реализации: `MLStrategy`, `StatAnalysisStrategy`

#### e) **Command Pattern**
- `Command` интерфейс
- Реализации: `AnalysisCommand`, `ResponseCommand`, `SolutionCommand`

### 5. **Модель данных (Domain Model)**

#### Основные сущности:
- `Request` - речевые запросы
- `ResponseEntity` - ответы на запросы
- `Sensor` - датчики
- `SensorData` - данные с датчиков
- `Solution` - решения/анализы
- `Sound` - звуковые данные
- `Language` - перечисление языков

### 6. **Особенности реализации**

#### a) **Кэширование**
- Используется EhCache с конфигурацией в `ehcache.xml`
- `CacheMonitorService` для управления кэшем

#### b) **Транзакционность**
- `TransactionManager` для управления транзакциями
- JTA транзакции через JPA

#### c) **Безопасность**
- `AuthFilter` для аутентификации
- `AuthServlet` для управления сессиями

#### d) **REST API**
- Ресурсы для всех основных сущностей
- Поддержка CRUD операций
- Специализированные endpoints для анализа данных

### 7. **Веб-интерфейс**
- **Servlets**: `ViewServlet`, `AuthServlet`
- **HTML шаблоны**: разделены по функциональным областям
- **Фильтры**: кодировка, аутентификация

### 8. **Конфигурация**
- **persistence.xml** - настройки JPA
- **web.xml** - конфигурация сервлетов и фильтров
- **beans.xml** - активация CDI
- **context.xml** - настройки контекста

### 9. **Сборка и развертывание**
- **Maven/Gradle** структура (предположительно)
- **WAR** артефакт для развертывания на сервере приложений
- Тесты в отдельной директории

## Преимущества архитектуры

1. **Масштабируемость**: Четкое разделение слоев позволяет легко добавлять новую функциональность
2. **Поддерживаемость**: Каждый слой имеет свою четкую ответственность
3. **Тестируемость**: Сервисы и репозитории можно легко тестировать изолированно
4. **Гибкость**: Использование паттернов позволяет легко изменять реализацию отдельных компонентов
5. **Безопасность**: Многоуровневая архитектура с фильтрами безопасности
